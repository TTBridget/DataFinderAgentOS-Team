##舆情大屏（安全监控与预警）功能需求
1. 数据接入：
   - 实时或定时获取以下三类数据（从现有数据表中读取）：
     - 用户会话记录（`chat_session` 表）
     - 用户对话内容（`chat_message` 表，含提问和回答）
     - 瞭望采集数据（`collected_data` 表）
   - 数据展示需包含时间范围筛选器（如近24小时、近7天、自定义区间）。

2. 敏感词检测与预警：
   - 预设一个敏感词库（初始数据可硬编码，后续允许在系统设置中维护）。
   - 对对话内容和采集内容进行扫描，命中敏感词时：
     - 在界面顶部以**红色警示条**或**弹窗通知**提醒管理员，并高亮显示违规记录。
     - 同时通过系统消息（可在 `user` 表中增加字段或新建 `notification` 表）向触发用户反馈预警信息，内容如“您的发言包含敏感词汇，请遵守社区规范”。
   - 预警列表需展示：触发时间、用户姓名、触发词、所属会话/采集源、处理状态（待处理/已忽略/已反馈）。

3. AI 自主风险分析：
   - 调用现有 `services/intent_engine.py` 中的 AI 能力（或新增分析服务），对上述数据进行综合分析，输出：
     - 整体风险等级（高/中/低），以仪表盘或色块展示。
     - 热点话题排行（基于词频或简单聚类），以横向条形图展示。
     - 敏感词触发趋势（按小时/天），以折线图展示。
   - 上述分析结果需在舆情大屏中以 **ECharts** 图表呈现（ECharts 已在 `static/dist` 中有引用）。

4. 界面布局（参考现有 `admin/index.html` 风格，浅色系，科技风，炫酷感）：
   - 顶部：统计卡片（总会话数、总对话数、采集总数、待处理预警数）。
   - 中左：风险等级分布（饼图/环形图）。
   - 中右：热点话题词云（使用 WordCloud 插件）。
   - 下左：敏感词触发趋势（折线图）。
   - 下右：最新预警列表（表格，可点击行弹出详情弹窗，显示完整原文）。
###前端实现要求
- 使用 LayUI 的栅格系统进行布局，图表容器使用 `div` 并设置固定宽高。
- 通过 AJAX 调用上述 API 获取数据并渲染 ECharts 图表（ECharts 通过 CDN 或本地 `static/dist/echarts` 引入）。
- 预警列表使用 LayUI 的表格组件，支持分页和状态切换。
- 点击预警行弹出 `layer.open` 详情弹窗，显示原文及操作按钮（“标记已处理”、“发送反馈”）。
