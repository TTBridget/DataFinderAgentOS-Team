# DataFinderAgentOS 开发约束

## 技术栈约束

### 后端开发
1. **必须使用 Tornado Web 框架**，不得引入其他 Web 框架（如 Flask、Django）
2. **数据库必须使用 SQLite3**（Python 内置，零依赖），不得引入其他数据库系统
3. **必须使用 Python 3**，兼容 Python 3.11 及以上版本
4. **ORM 约束**：不得引入 ORM 框架（如 SQLAlchemy），必须使用原生 SQL + Repository 模式
5. **实时通信**：支持 WebSocket 和 SSE（Server-Sent Events）用于实时通信
6. **大模型集成**：支持 OpenAI API 集成，用于大模型对话功能

### 前端开发
1. **优先使用 Layui 框架**（v2.x）作为主要 UI 组件库，文档参考：https://layui.dev/docs/2/
2. **Bootstrap 作为补充**：用于特殊样式需求或其他界面需求
3. **UI 组件本地化**：所有组件包已本地化在 `app/static/dist/` 目录下（包含 Layui 和 Bootstrap）
4. **模板引擎**：必须使用 Tornado 内置模板引擎
5. **静态资源**：必须放置在 `app/static/` 目录下
6. **模板文件**：必须放置在 `app/templates/` 目录下

### 界面设计要求
1. **响应式设计**：界面必须完全响应式，适配不同屏幕尺寸
2. **沉浸式风格**：采用现代化的沉浸式设计风格
3. **自适应模式**：同时支持移动端和 PC 端的完美适配

## 架构约束

### 目录结构约束
- **控制器层**：必须放在 `app/controllers/` 目录，继承 `BaseHandler`
- **模型层**：必须放在 `app/models/` 目录，使用 Repository 模式
- **静态资源**：`app/static/` 目录包含 `css/`、`js/`、`dist/` 子目录
- **模板文件**：用户端模板在 `app/templates/`，管理员模板在 `app/templates/admin/`

### 代码组织约束
1. **MVC 架构必须严格执行**：
   - Model 层只负责数据访问，不包含业务逻辑
   - Controller 层负责请求处理、参数验证、业务逻辑调度
   - View 层只负责页面渲染，不包含业务逻辑

2. **Repository 模式**：所有数据库操作必须通过 Repository 类的静态方法完成
3. **BaseHandler 继承**：所有 Controller 必须继承 `app.controllers.base.BaseHandler`

## 安全约束

### OWASP Top 10 安全策略
1. **注入攻击防护**（SQL、NoSQL、OS 命令等）
   - 所有 SQL 查询必须使用参数占位符（`?`），禁止字符串拼接
   - 对用户输入进行严格验证和过滤

2. **认证与会话管理安全**
   - 使用 `@tornado.web.authenticated` 装饰器保护需要登录的页面
   - 使用 `get_secure_cookie` 和 `set_secure_cookie` 管理登录状态
   - 设置合理的会话过期时间
   - 密码必须使用 salt + pbkdf2_hmac 进行哈希处理，不得存储明文密码

3. **敏感数据泄露防护**
   - 敏感数据（密码、密钥等）在传输和存储时必须加密
   - 不要在日志中记录敏感信息
   - 错误信息不应该暴露系统内部细节

4. **XXE（XML 外部实体）防护**
   - 禁用不安全的 XML 解析器功能
   - 如果必须处理 XML，使用安全的解析方式

5. **访问控制失效防护**
   - 实现严格的权限验证机制
   - 每个受保护的资源都必须验证用户权限
   - 禁止直接通过 URL 访问受保护资源

6. **安全配置错误防护**
   - 所有配置存放在 `config/` 目录下
   - 默认配置应该是安全的
   - 定期更新和检查安全配置

7. **XSS（跨站脚本）防护**
   - 用户输入输出必须进行适当的转义处理
   - 设置适当的 Content-Security-Policy 头

8. **不安全的反序列化防护**
   - 避免不安全的反序列化操作
   - 验证和清理反序列化的数据

9. **使用含有已知漏洞的组件防护**
   - 定期检查并更新依赖包
   - 移除未使用的依赖和功能
   - 使用官方和受信任的源

10. **日志和监控不足防护**
    - 记录所有关键操作和安全事件
    - 监控异常活动
    - 确保日志中不包含敏感信息

### 其他安全约束
1. **CSRF 防护**：所有 POST 请求必须使用 Tornado 的 xsrf_cookies 机制
2. **配置管理**：所有配置必须存放在 `config/` 目录下

## 代码风格约束

1. **缩进**：使用 Tab 缩进（4 个空格宽度）
2. **命名规范**：
   - 类名：大驼峰命名（如 `UserRepository`）
   - 函数/方法：小写下划线命名（如 `get_user_by_username`）
   - 常量：全大写下划线命名
3. **注释规范**：公共 API 和复杂逻辑必须有清晰的注释
4. **错误处理**：合理使用异常处理，返回适当的 HTTP 状态码

## 开发环境约束

1. **虚拟环境**：必须使用项目根目录下的 `venv/` 虚拟环境
2. **运行端口**：默认运行在 10010 端口
3. **数据库文件**：必须使用 `database/finderos.db`，不得更改路径

## 其他约束

1. **最小依赖原则**：尽量不引入新的第三方依赖包
2. **向后兼容**：修改代码时保持向后兼容性
3. **测试覆盖**：重要功能必须有对应的测试用例，放在 `test/` 目录下
